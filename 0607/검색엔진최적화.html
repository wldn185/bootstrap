검색엔진최적화(SEO, Search Engine Optimized) - meta

  • !DOCTYPE html : 문서 버전을 의미, html5
  • lang="ko" : 문서안에 한글이 우선적으로 들어가 있다는 의미(주로 사용하는 언어 명시)
  • charset="UTF-8" : charset(문자언어코드), 'UTF-8'(국제표준코드) 'euc-kr'(한글표준코드)
  • meta : 문서의 '내용'이나 '저자', '날짜정보', '키워드' 등을 나타내고자할 때 사용.

Meta태그는 웹서버와 브라우저간의 상호 교환되는 정보를 정의하는데 사용하며, 웹페이지가 담고 있는 콘텐츠가 아닌 웹페이지 자체의 정보를 명시하기 위한 목적으로 사용되는 html태그를 의미한다.

이러한 웹페이지의 메타정보는 주로 html의 meta요소를 통해 마크업을 하며, html문서 내에서 head요소아래 배치되므로 일반 유저가 보게 되는 웹페이지 디자인에는 전혀 영향을 미치지 않고, 문서가 어떤 내용을 담고 있고, 문서의 키워드는 무엇이며, 누가 만들었는지 등의 문서 자체의 특성을 담고 있다.

반면 검색엔진(naver, daum, google, nate...)과 같은 기계들이 웹페이지를 읽어야 할 때는 메타 태그의 내용들이 해당 서비스에서 어떻게 표시될지를 결정하는 매우 핵심적인 요소가 된다.

1. SEO - TITLE

검색엔진은 주기적으로 웹사이트를 방문하여 각 웹페이지가 어떤 콘텐츠를 제공하고 있는지를 파악하고 수집하여 데이터베이스 저장을 하는데, 점점 지능적(AI)으로 진화하고 있다. 또한 어느정도까지는 스스로 웹페이지의 구조와 내용을 파악할 수 있다.그럼에도 불구하고 메타태그를 통해 웹페이지에 대한 정보를 좀더 주도적으로 명시한다면 SEO(검색엔진 최적화)측면에서 유리하다.

예) 메인페이지인 경우 - 스타벅스에 오신 것을 환영합니다. starbucks korea, 취항공감 라이브쇼핑 - cj

예) 메인페이지가 아닌 경우(서브페이지) - 스타벅스 - 오시는길, 살림비책 6호

2. SEO - META

웹사이트 제목만 예외적으로 title요소를 사용하고, 그 밖에 다른 웹페이지의 메타정보는 meta요소를 사용하여 표현한다.

meta요소를 사용할 때는 name속성을 통해 메타정보의 이름을 명시하고 content속성을 통해 메타 정보의 이름과 내용을 명시한다.

META태그의 3가지 속성
1. html=equiv='항목명'
웹브라우저가 서버에 명령을 내리는 속성으로 name속성을 대신하여 사용될 수 있으며, html문서가 응답 헤더와 함께 웹 서버로부터 웹 브라우저에 전송되었을 때만 의미
2. content='정보값'
meta정보의 내용을 지정.
3. name='정보이름'
몇 개의 meta정보의 이름을 지정할 수 있으며 지정하지 않으면 http=equiv와 같은 기능을 함

meta태그의 종류

  1. meta name="keywords" content="여성의류, 웹표준" : 검색엔진에 의해 검색되는 단어를 지정
  2. meta name="Description" content="신상의류, 자신의 스타일에 맞는 의상 추천" : 검색결과에 표시되는 문자를 지정
  3. meta name="Robots" content="noindex, nofollow" : 검색로봇 제어
    • All(기본값) : 'index, follow'
    • None = 'noindex, nofollow'
    • Index : 그 페이지를 수집 대상으로 함.
    • Follow : 그 페이지를 포함해 링크가 걸린 곳을 수집 대상으로 함.
    • Noindex : 그 페이지를 수집대상에서 제외
    • Nofollow : 그 페이지를 포함하여 링크가 걸린 곳을 수집 대상으로 하지 않습니다.
  5. meta http-equiv="Content-Type" content="text/html; charset=utf-8": 문자 코드의 종류를 설정
  6. meta name="Date" content="2023-01-19T07:45:37+09:00" : 작성날짜(제작일)
  7. meta http-equiv="X-UA-Compatible" content="IE=edge" : 브라우저 호환성을 지정
  8. meta http-equiv="Subject" content="웹 표준을 위한 사이트" : 홈페이지 주제
  9. meta http-equiv="Title" content="웹 표준" : 홈페이지의 제목
  10. meta http-equiv="Author" content="name" : 작성자
  11. meta http-equiv="Order Agent" content="name" : 책임자
  12. meta http-equiv="Distribution" content="name" : 배포자
  13. meta http-equiv="Copyright" content="copyright@dothome" : 저작권
  14. meta http-equiv="Build" content="date" : 제작일
  15. meta http-equiv="Last-Modified" content="Mon,20 Jul 2016 19:30:30" : 최종수정일
  16. meta http-equiv="Refresh" content="60" : 지정한 시간마다 새로고침
  17. meta http-equiv="Refresh" content="5;url=주소" : 입력한 주소로 지정시간 후 자동 연결
  18. meta http-equiv="Page-Enter" content="revealtrans(Duration=1, Transition=12)" : 페이지 장면전환 효과

SEO(검색엔진 최적화)태그 - Open Graph

요즘에는 SNS와 메신저를 통해 웹페이지의 링크를 공유하는 일이 잦아지면서 메타 태그가 좀 더 넓은 분야에서 활용되고 있음. 대표적인 사례로 페이스북이나 카카오톡, 슬랙(Slack)과 같은 앱에서 웹페이지의 링크만 입력하더라도 해당 웹페이지의 콘텐츠의 미리보기를 카드 형태로 나타나는 것을 볼 수 있음.

일반적으로 이러한 서비스들은 Open Grahp 프로토콜이라고 불리는 업계 표준을 따라서 '웹페이지 콘텐츠 미리보기'를 지원하고 있음. 따라서 우리도 Open Graph 프로토콜에서 정의하고 있는 메타 태그를 적절히 사용하면 콘텐츠미리보기에 원하는 내용을 표시되도록 어느정도 제어할 수 있음

Open Graph 태그의 이름은 og:로 시작한다.

meta name="og:site_name" content="사이트명" : 사이트제목

meta name="og:title" content="페이지 제목" : 웹페이지의 제목

meta name="og:description" content="페이지 설명" : 웹페이지의 상세 설명

meta name="og:type" content="webpage" : 웹페이지의 유형

meta name="og:url" content="http://design2010.dothome.co.kr" : 웹페이지의 주소

meta name="og:image" content="http://이미지 경로" : 웹페이지의 썸네일 이미지 주소

검색엔진이 검색할 수 있도록 제어하는 robots.txt파일 작성법
